# Техническое задание: Telegram-бот «Вопросы шейху»

## 1. Назначение

Telegram-бот предназначен для исламского проекта «Вопросы шейху». Обычный пользователь отправляет вопрос по исламской теме, а шейх/администратор получает вопрос в панели, отвечает на него и при необходимости закрывает обращение.

## 2. Роли

### Пользователь

Может:

- выбрать язык интерфейса;
- задать вопрос шейху;
- выбрать категорию вопроса;
- отправить текст, фото, документ, видео или голосовое сообщение;
- смотреть свои вопросы;
- получить ответ шейха в боте.

### Шейх/администратор

Определяется по Telegram ID из переменной `ADMIN_IDS`.

Может:

- открыть отдельную панель шейха;
- видеть список новых, отвеченных и закрытых вопросов;
- открывать карточку вопроса;
- отвечать пользователю текстом или медиа;
- закрывать вопрос;
- смотреть статистику;
- менять язык панели.

У шейха/админа нет пользовательской кнопки «Задать вопрос шейху», потому что он сам является отвечающей стороной.

## 3. Языки

Поддерживаются:

- Русский;
- English;
- العربية.

Выбранный язык сохраняется в базе данных SQLite.

## 4. Категории вопросов

- Акыда / основы веры;
- Намаз, пост, закят, хадж;
- Фикх / бытовые вопросы;
- Семья и брак;
- Работа, бизнес и деньги;
- Коран и хадисы;
- Нравственность и поведение;
- Другое.

## 5. Статусы вопросов

- `open` — новый вопрос;
- `answered` — шейх ответил;
- `closed` — вопрос закрыт.

## 6. Основной сценарий пользователя

1. Пользователь запускает бота командой `/start`.
2. Бот показывает пользовательское меню.
3. Пользователь нажимает «Задать вопрос шейху».
4. Бот просит выбрать категорию.
5. Пользователь отправляет вопрос.
6. Бот создаёт обращение и уведомляет шейха/админов.
7. Пользователь ждёт ответ.
8. Ответ шейха приходит в бот.

## 7. Основной сценарий шейха

1. Шейх запускает `/start` или `/panel`.
2. Бот показывает панель шейха.
3. Шейх выбирает «Новые вопросы».
4. Шейх открывает карточку вопроса.
5. Шейх нажимает «Ответить».
6. Шейх отправляет ответ.
7. Бот пересылает ответ пользователю.
8. Вопрос получает статус `answered`.
9. Шейх может закрыть вопрос.

## 8. Технический стек

- Python 3.11+;
- aiogram 3.x;
- SQLite;
- aiosqlite;
- python-dotenv;
- tzdata для корректной работы часовых поясов на Windows.

## 9. Переменные окружения

```env
BOT_TOKEN=токен_бота_от_BotFather
ADMIN_IDS=telegram_id_шейха
DATABASE_PATH=data/support_bot.db
```

Несколько админов:

```env
ADMIN_IDS=111111111,222222222,333333333
```

## 10. Команды

### Пользователь

```text
/start — открыть меню
/new — задать вопрос
/my — мои вопросы
/language — сменить язык
/help — помощь
```

### Шейх/админ

```text
/start — открыть панель шейха
/panel — открыть панель шейха
/answer 123 — ответить на вопрос #123
/close 123 — закрыть вопрос #123
/language — сменить язык панели
/help — помощь для шейха
/cancel — отменить ввод ответа
```

## Дополнительные требования, реализованные в последней версии

1. Обычный пользователь не должен открывать карточку вопроса через кнопку «Открыть вопрос».
2. В разделе «Мои вопросы» пользователь видит только краткий список обращений.
3. Админ/шейх должен видеть полную историю вопроса: вопрос, сообщения, ответы, даты, автора вопроса и автора каждого ответа.
4. Все админы должны видеть ответы других админов/шейхов.
5. В панели шейха должен быть отдельный раздел «История ответов» с последними ответами по всем вопросам.

## Обновление: исправление обычного пользователя и тесты

Добавлен фильтр `NonAdminFilter`, который гарантирует, что пользовательский роутер обрабатывает только обычных пользователей, а администраторы/шейхи обслуживаются отдельным админским роутером. Это снижает риск конфликтов между админскими и пользовательскими обработчиками.

Добавлен набор автотестов:

1. Парсинг списка админов из `.env`.
2. Отсутствие пользовательской кнопки «Открыть вопрос».
3. Корректное разделение админа и обычного пользователя фильтром.
4. Сохранение истории ответов админов в SQLite.

## Дополнение к ТЗ: история ответов и меню команд

### История ответов админов/шейхов

В панели шейха должна быть доступна общая история ответов. Все администраторы видят ответы всех администраторов. Для каждой записи отображается:

- номер вопроса;
- тема вопроса;
- текущий статус обращения;
- дата создания вопроса;
- пользователь, задавший вопрос;
- текст исходного вопроса;
- администратор/шейх, давший ответ;
- дата ответа;
- текст ответа или тип вложения.

История должна иметь пагинацию.

### Telegram menu button

Бот должен программно регистрировать команды Telegram, чтобы пользователь мог открыть меню команд рядом с полем ввода и не потеряться в интерфейсе.

## Дополнение к ТЗ: история ответов без обрезки

Для шейхов/администраторов история ответов должна быть реализована в формате:

- общий список ответов с пагинацией и короткими превью;
- отдельная полная карточка каждого ответа;
- в полной карточке вопрос и ответ не должны обрезаться;
- если текст превышает лимит Telegram, бот делит его на несколько сообщений;
- карточка должна быть пригодна для пересылки в группу.


## Требование к голосовым сообщениям

История ответов должна корректно работать с голосовыми сообщениями:

1. Нельзя показывать технический текст вида `[ContentType.VOICE]`.
2. В краткой истории должна отображаться понятная пометка `🎙 Голосовое сообщение`.
3. В полной карточке ответа должна быть информация, что оригинал вложения доступен отдельной кнопкой.
4. Админ должен иметь возможность получить оригинальное голосовое сообщение из истории и переслать его в группу.
5. Для голосовых вопросов и голосовых ответов должны сохраняться `content_type` и `file_id`.
