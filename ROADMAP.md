# Roadmap: роли шейха, админа и публикации

## Цели

- [x] Разделить админов на две роли: простой интерфейс шейха и расширенный интерфейс обычного админа.
- [x] Убрать кнопку закрытия вопроса из админских карточек, чтобы вопрос нельзя было случайно закрыть.
- [x] Для шейха оставить минимальный интерфейс: входящие вопросы, кнопка ответа, списки отвеченных и неотвеченных вопросов.
- [x] Для обычного админа сохранить текущие рабочие возможности и добавить просмотр ответов шейха перед публикацией.
- [x] Добавить статус опубликованных ответов и скрывать опубликованные из списка ожидающих публикации.
- [x] Добавить публикацию одобренного ответа в Telegram-канал в формате, близком к приложенному скриншоту.
- [x] Вынести ссылку/идентификатор канала в `.env`.
- [x] Покрыть новую логику тестами и прогнать полный набор тестов.
