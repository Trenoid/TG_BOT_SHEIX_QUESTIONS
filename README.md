# Вопросы шейху — Telegram-бот

Бот для исламского проекта: пользователь задаёт вопрос шейху, выбирает тему и язык интерфейса, шейх или администратор отвечает прямо в боте, а ответы публикуются в Telegram-канал.

## Что есть в версии Sheikh Panel

- Отдельное меню для обычного пользователя.
- Две роли staff: простой интерфейс шейха через `SHEIKH_IDS` и расширенная панель администратора через `ADMIN_IDS`.
- У админа больше нет кнопки «Задать вопрос шейху».
- В карточках больше нет кнопки «Закрыть», чтобы вопрос не закрыли случайно.
- Шейх видит только два раздела: неотвеченные и отвеченные вопросы.
- Новый вопрос приходит шейху в простом виде: сам вопрос и кнопка «Ответить».
- Администратор видит новые, отвеченные, опубликованные и закрытые вопросы.
- Администратор может открыть ответ в формате будущей публикации и отправить его в канал.
- Текстовые ответы шейха автоматически публикуются в канал без ручного подтверждения админа, только если вопрос тоже был обычным текстом.
- Добавлен статус `published` для опубликованных ответов.
- Поддержка нескольких шейхов и админов через `SHEIKH_IDS` и `ADMIN_IDS`.
- 3 языка интерфейса для пользователей: русский, английский, арабский.
- Смена языка через кнопку и команду `/language`.
- Исламские категории вопросов:
  - Акыда / основы веры
  - Намаз, пост, закят, хадж
  - Фикх / бытовые вопросы
  - Семья и брак
  - Работа, бизнес и деньги
  - Коран и хадисы
  - Нравственность и поведение
  - Другое
- Пользовательские вопросы принимаются текстом, фото или видео с подписью. Голосовые вопросы запрещены.
- Ответы шейха/админа поддерживают текст, фото, документы, видео и голосовые сообщения.
- Хранение истории в SQLite.

## Команды пользователя

```text
/start — открыть меню
/new — задать вопрос
/my — мои вопросы
/language — сменить язык
/help — помощь
```

## Команды шейха

```text
/start — открыть меню вопросов
/panel — открыть меню вопросов
/answer 123 — ответить на вопрос #123
/help — помощь для шейха
/cancel — отменить ввод ответа
```

## Команды администратора

```text
/start — открыть панель администратора
/panel — открыть панель администратора
/answer 123 — ответить на вопрос #123
/language — сменить язык панели
/help — помощь администратора
/cancel — отменить ввод ответа
```

## Установка

### 1. Распакуйте архив

Перейдите в папку проекта:

```powershell
cd tg_support_bot_sheikh
```

### 2. Создайте виртуальное окружение

```powershell
python -m venv venv
```

### 3. Активируйте окружение

PowerShell может блокировать запуск скриптов. Самый простой вариант:

```powershell
cmd /k .\venv\Scripts\activate.bat
```

Или без активации можно запускать так:

```powershell
.\venv\Scripts\python.exe -m pip install -r requirements.txt
.\venv\Scripts\python.exe -m app.main
```

### 4. Установите зависимости

```powershell
pip install -r requirements.txt
```

### 5. Создайте `.env`

Скопируйте пример:

```powershell
copy .env.example .env
```

Заполните файл `.env`:

```env
BOT_TOKEN=токен_бота_от_BotFather
ADMIN_IDS=691367615
SHEIKH_IDS=123456789
DATABASE_PATH=data/support_bot.db
PUBLICATION_CHANNEL=https://t.me/your_channel
```

Несколько админов или шейхов можно указать через запятую:

```env
ADMIN_IDS=691367615,123456789,987654321
SHEIKH_IDS=111111111,222222222
```

Если один и тот же Telegram ID указан и в `ADMIN_IDS`, и в `SHEIKH_IDS`, бот считает его обычным администратором и показывает расширенную панель.

`PUBLICATION_CHANNEL` принимает публичную ссылку `https://t.me/channel`, username `@channel` или числовой id канала вида `-100...`. Бот должен быть администратором канала.

Публикация в канал отправляется в таком формате:

```text
Ответы на вопросы | Шейх Абдул-Малик Хайров

ВОПРОС ❓ №14:
текст вопроса

ОТВЕТ✅:
🎙 Голосовое сообщение

[Орфография и пунктуация автора сохранены]

✅ Ответы Шейха
```

Текст «Ответы Шейха» в последней строке содержит ссылку на канал из `PUBLICATION_CHANNEL`.

## Как работает

### Пользователь

1. Открывает `/start`.
2. Выбирает язык.
3. Нажимает «Задать вопрос шейху».
4. Выбирает категорию.
5. Отправляет вопрос.
6. Получает ответ от шейха в этом же боте.

### Шейх

1. Открывает `/start` или `/panel`.
2. Видит только «Неотвеченные вопросы» и «Отвеченные вопросы».
3. Открывает неотвеченные вопросы.
4. Нажимает на нужный вопрос и видит текст вопроса.
5. Нажимает «Ответить».
6. Пишет ответ текстом или отправляет медиа.
7. Ответ приходит пользователю.

### Администратор

1. Открывает `/start` или `/panel`.
2. Видит расширенную панель администратора.
3. Открывает «Отвеченные вопросы» — там ответы, которые ещё не опубликованы.
4. Открывает конкретный ответ и видит его в формате публикации для канала.
5. Нажимает «Опубликовать», если бот может опубликовать такой ответ. Такая же кнопка появляется сразу под подтверждением, если администратор сам ответил на вопрос.
6. Бот отправляет публикацию в канал из `PUBLICATION_CHANNEL`, а вопрос получает статус `published`.
7. Опубликованные ответы доступны через «Опубликованные ответы».

Автопубликация работает только для связки «текстовый вопрос + текстовый ответ». Публикация через кнопку работает только для «текстовый вопрос + текстовый или голосовой ответ». Если ответ голосовой, бот публикует его одним сообщением с caption; из-за лимита Telegram длинный вопрос в таком caption может быть сокращён. Если в вопросе есть фото/видео или в ответе есть что-то кроме текста/голосового, админ публикует вручную и нажимает «Отметить как опубликованный», чтобы убрать ответ из очереди.

## Частые ошибки

### TelegramUnauthorizedError: Unauthorized

Проверьте `BOT_TOKEN` в `.env`. Это должен быть именно токен от BotFather.

### No module named tzdata

В этой версии `tzdata` уже добавлен в `requirements.txt`. Выполните:

```powershell
pip install -r requirements.txt
```

### PowerShell запрещает активацию venv

Запустите:

```powershell
cmd /k .\venv\Scripts\activate.bat
```

или используйте Python напрямую из окружения:

```powershell
.\venv\Scripts\python.exe -m app.main
```

## Исправление пользовательского отклика

В этой версии пользовательские обработчики дополнительно защищены фильтром `NonAdminFilter`: обычные пользователи всегда попадают в пользовательское меню, а админы/шейхи — только в панель шейха. Это исправляет ситуацию, когда у админа всё работает, а обычный пользователь не получает ответ на `/start` или кнопки.

## Тесты

В проект добавлены базовые тесты:

- проверка парсинга `ADMIN_IDS`;
- проверка парсинга `SHEIKH_IDS`/канала публикации;
- проверка, что у пользователя нет кнопки открытия карточки вопроса;
- проверка, что у админа нет кнопки закрытия вопроса;
- проверка минимальных клавиатур шейха;
- проверка фильтра, который разделяет админов и обычных пользователей;
- проверка истории ответов админов в базе данных.

## Обновление: полная история ответов для админов

В админской панели раздел **📜 История ответов** теперь работает в два уровня:

1. В общем списке показываются только краткие превью вопроса и ответа, чтобы сообщение не обрезалось Telegram.
2. Под списком есть кнопки **📄 Полный текст: вопрос #... / ответ #...**.
3. По кнопке открывается полная карточка с:
   - номером вопроса;
   - темой;
   - статусом;
   - языком пользователя;
   - кем задан вопрос;
   - временем вопроса;
   - полным текстом вопроса;
   - кем отвечено;
   - временем ответа;
   - полным текстом ответа.

Такую полную карточку удобно пересылать в группу. Если карточка слишком длинная, бот автоматически отправит её несколькими частями.


## Обновление: голосовые сообщения в истории

В истории ответов голосовые сообщения больше не отображаются как `[ContentType.VOICE]`.
Бот показывает понятную пометку `🎙 Голосовое сообщение`, а в полной карточке ответа появляются кнопки:

- `📎 Отправить оригинал вопроса` — для старых вопросов с вложениями, если они остались в базе;
- `🎙/📎 Отправить оригинал ответа` — если ответ шейха был голосовым/файлом/фото/видео.

Админ может нажать кнопку, получить оригинальное голосовое отдельным сообщением и переслать его в группу.
